# Swin-Base 特征蒸馏修改说明

## 目标

modified by zhoujiwen：新增一条不依赖 `dino_head` / `ibot_head` 的特征蒸馏训练路径，用本地 DINOv3 ViT-L/16 backbone 权重蒸馏 Swin-Base student。

## Teacher 与 Student

Teacher checkpoint：

```text
checkpoints/dinov3_vitl16_pretrain_lvd1689m-8aa4cbdd.pth
```

该 checkpoint 已查验：

```text
模型类型：DINOv3 ViT-L/16 backbone
输出维度：1024
patch size：16
是否包含 dino_head / ibot_head：不包含
```

Student：

```text
arch: swin_base
patch size: 4
最终输出维度: 1024
```

对于 64x64 输入：

```text
Teacher ViT-L/16: 64 / 16 = 4x4 patch tokens
Student Swin-Base: 64 / 4 = 16x16 初始 patch tokens，stage4 后为 2x2 tokens
```

当前实现按你的要求使用 Swin stage4/final patch tokens，并将 `2x2` 双线性上采样到 `4x4`，再与 teacher 的 `4x4` patch tokens 计算特征蒸馏 loss。

## Loss

由于 teacher checkpoint 不含训练 head，当前不使用原始 DINO/iBOT prototype loss，而使用特征空间 loss：

```text
total_loss =
  cls_feature_loss
+ patch_feature_loss
+ masked_patch_feature_loss
+ KoLeo loss
```

说明：

```text
cls_feature_loss：student CLS 对齐 teacher CLS
patch_feature_loss：student stage4 patch 上采样到 4x4 后对齐 teacher patch
masked_patch_feature_loss：沿用原 iBOT mask 策略，只在 masked patch 位置计算特征 loss
KoLeo loss：作用在 student CLS 上
```

Gram loss 已在源码中硬禁用。

## 一键训练

不需要手动输入长命令，Ubuntu A40 服务器上直接运行：

```bash
bash scripts/run_swin_base_feature_distill.sh
```

脚本会检查 CUDA，如果未检测到 CUDA，会直接退出，避免误在 CPU 上训练。
脚本也会检查 teacher checkpoint 和数据目录是否存在；任一路径不存在都会在训练启动前退出。

默认使用 1 张 GPU：

```bash
CUDA_VISIBLE_DEVICES=0 bash scripts/run_swin_base_feature_distill.sh
```

如需多卡调试：

```bash
CUDA_VISIBLE_DEVICES=0,1 NPROC_PER_NODE=2 bash scripts/run_swin_base_feature_distill.sh
```
