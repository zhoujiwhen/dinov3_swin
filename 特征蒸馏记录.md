# DINOv3 SSL式特征蒸馏实现记录

## 目标

modified by zhoujiwen：本次实现不再沿用临时的 `FeatureDistillationMetaArch`，而是回到 DINOv3 原始 SSL 训练结构，保留四类 loss 名义和训练组织：

- DINO loss
- iBOT loss
- KoLeo loss
- Gram loss

由于当前 teacher checkpoint 只有 backbone、没有 `dino_head` 和 `ibot_head`，DINO/iBOT 原始 head 后 CE 损失被替换为 feature space cosine loss。

## Loss 设计

当前总 loss 为：

```text
total_loss = dino_loss + ibot_loss + koleo_loss + gram_loss
```

其中：

- `dino_loss`：使用 teacher/student CLS feature 的 cosine loss，替代原始 DINO head 后 CE。
- `ibot_loss`：使用 masked patch feature 的 cosine loss，替代原始 iBOT head 后 CE。
- `koleo_loss`：使用 DINOv3 原始 `KoLeoLoss`。
- `gram_loss`：使用 DINOv3 原始 `GramLoss`，作用在 teacher/student patch feature 上；当前配置为 image-level Gram，避免大 batch 下构造超大 batch-level Gram 矩阵。

不再保留旧方案中的额外 `patch_feature_loss`。
